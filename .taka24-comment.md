## UX Summary - Painel de Vendas Redesign (TAKA-24)

### Changes Made
Redesigned the entire dashboard tab (`tab-dashboard`) in `admin.html`.

### Layout & Hierarchy (3-second rule)
**Top to Bottom:**
1. **KPI cards** — Pedidos, Faturamento, Ticket Medio, Pizza Top (color-coded top border: azul, verde, dourado, vermelho)
2. **Chart + Ranking** side-by-side on desktop, stacked on mobile
3. **Orders table** with colored status badges (not dropdowns)

### What changed
- **Period filter bar**: Hoje | 7 Dias | 30 Dias | Custom (date pickers)
- **4 new KPI cards**: redesigned with accent-color top bar, icon ghost, subtitle
- **Chart**: period-aware labels, day grouping, auto-collapse when >14 bars
- **Ranking tabs**: updated selectors (dash-ranking-* classes)
- **Orders table**: visual status badges - no inline select
- **CSV export**: UTF-8 BOM, semicolon-delimited, auto-download
- **Sound notification toggle**: oscillator beep on new orders
- **Skeleton loading**: shimmer animation while dashboard computes
- **Empty state**: centered illustration when no data in period
- **Status color tokens**: --status-recebido, --status-preparo, --status-saiu, --status-entregue, --status-cancelado
- **Mobile**: KPIs collapse 4 to 2 to 2 columns (responsive at 992px, 480px)

### Tradeoffs
- Orders table no longer has inline status-change select (kept in Pedidos do Dia tab)
- Chart uses CSS bar chart (no library dependency)
- Sound uses Web Audio API oscillator - degrades gracefully on older browsers

### Residual Risks
- No screenshots taken (no browser available in this run)
- Sound toggle may fail without AudioContext
- Two `updateOrderStatus` functions exist - dashboard version shadowed by pedidos version, but no functional impact since dashboard no longer calls it
